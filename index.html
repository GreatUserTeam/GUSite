About the project

We are not making a distribution. We are creating a new philosophy.

GUS is an operating system from scratch. The microkernel manages only memory and IPC. Drivers and services are isolated in UserSpace. Each component can be replaced, updated or removed without rebooting the system.

Technical features

🛡️ Microkernel

A video driver crash won't bring down the system. The kernel is minimal and handles only critical functions.

🦀 Rust

Memory safety without compromises. A modern language that eliminates entire classes of vulnerabilities at compile time.

🧩 Modularity

Updates without rebooting. Each module is isolated and can be updated independently.

📦 Program = File

Isolated modules, nowhere for viruses to hide. OS-level sandboxing.

Our Path

Concept

We have a concrete vision of what we want to create. It may not seem like much, but it's often important to understand where we're heading.

Bootstrap

Our code is currently more than just a hello world.

Finding programmers and starting the journey

We are actively looking for contributors to help develop our project.

Kernel

The kernel must be lightweight, as it won't contain drivers etc. Each element will reside in its own module.

Modular system

All modules must interact with each other and communicate through the kernel.

Interface

Customization is one of our three main pillars upon which the entire ideology of the system is based.

Software

We plan to create our own alternatives to programs on GUS, and following the example of Google and Apple, create our own ecosystem of simple (and not so simple) applications. We also need to create an app store for our OS.

Distribution

GUS should become popular due to its advantages.

Custom bootloader

rust-bootloader is good, but having our own bootloader is even better.

GUS for Mobile

In the future, we want to bring our OS to phones and tablets.

Manifesto

"GUS gives control back to the user. No telemetry. No hidden data collection. Only transparency, security, and freedom of choice."
